# Skill Registry

> Scan the project's skill directories and produce a searchable index.
> The registry is the orchestrator's lookup table for delegating sub-agent work.
>
> Template convention: bare skill names â€” no plugin-namespace prefix.
> The `name:` field in each SKILL.md frontmatter MUST be a bare kebab-case name
> (e.g. `arch`, `component`) â€” not `plugin:arch` or `react-fractal:arch`.

---

## Pre-flight

1. Check if `.claude/skills/` exists.
2. Check if `.claude/skill-registry.md` already exists â€” if so, compare against
   scan results to identify new or removed skills.

---

## Scan Process

```
For each directory in .claude/skills/:
  1. Read SKILL.md frontmatter with bat
  2. Extract: name (first value after 'name:'), first line of description before "USE WHEN:"
  3. Extract: USE WHEN trigger phrases
  4. Record: relative path to SKILL.md

Sort alphabetically by skill name.
Confirm before writing .claude/skill-registry.md.
```

```bash
# Scan all SKILL.md files
fd SKILL.md .claude/skills/

# Read frontmatter of a specific file
bat .claude/skills/{name}/SKILL.md --line-range 1:10

# Check for name field
rg '^name:' .claude/skills/*/SKILL.md
```

## Integration with Orchestrator

When an orchestrator agent needs to delegate work, it reads this registry to
find matching skills by trigger and code context, then injects the SKILL.md
paths into the sub-agent prompt.

For this to work:
- Every skill must have `USE WHEN:` in its `description` frontmatter
- Trigger phrases must be specific and matchable
- The registry must be regenerated whenever a skill is added or modified

## DO vs DON'T

| DO                                          | DON'T                                 |
| ------------------------------------------- | ------------------------------------- |
| Re-run after adding or modifying any skill  | Let the registry go stale             |
| Commit the registry file to version control | Treat it as a cache â€” it IS the index |
| Flag skills with vague triggers             | Silently skip them                    |
| Confirm before writing the file             | Auto-write without asking             |
| Use bare names only (`arch`, `component`)   | Use plugin-namespaced names in `name:`|
| Use `fd`/`bat`/`rg` for scanning           | Use `find`/`cat`/`grep`               |
